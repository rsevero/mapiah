A few introductory instructions:

1. Avoid duplicated code. Extract reusable logic into separate methods or widgets.
2. All variable declarations should have explicit type definitions. Avoid using var when the type is not immediately obvious.

3. Variables should be declared as final whenever possible.
4. Avoid complex calculations in a single step. Break them down into intermediate variables with descriptive names to make the logic easier to understand.
5. In multiple condition decisions, wrap all comparisons that involve more than one element in parentheses for clarity.
6. Don´t use all caps on user text.
7. Don't worry about code formatting. This is automtically fixed on save.
8. Prefer named parameters; allow positional parameters only when there are at most two parameters and they have different types.
9. Run 'dart analyze' or similar on generated code.
